Legado — Limitações

  • Interface de texto só no servidor
  • Sem API — impossível integrar
  • Dados visíveis só no terminal
  • Sem paginação, filtros ou ordenação
  • Documento duplicado: erro genérico de banco

Moderno — Melhorias

  • API REST consumível por qualquer cliente
  • Filtragem por parâmetros: ?ativo=S
  • Respostas JSON estruturado
  • Acessível de qualquer lugar com JWT
  • 409 Conflict descritivo em documento duplicado

Legado — Bugs e Lacunas

  • Desconto hardcoded — sempre zero
  • Estoque nunca atualizado ao criar pedido
  • INSERT de cabeçalho fora de transação
  • Cancelar pedido já faturado sem validação
  • Status CONFIRMADO/FATURADO sem operação real

Moderno — Correções

  • Desconto operacional por item
  • Estoque decrementado/restituído automaticamente
  • @Transactional — operação 100% atômica
  • Máquina de estados: 422 em transição inválida
  • PATCH /confirmar e /faturar implementados

  • Extensível: adicionar filtros sem quebrar contrato
  • Acessível via curl, Postman, Python...

# Problema no Legado Solução no Moderno
1 Senha armazenada em texto puro BCrypt (fator 10) em toda persistência
2 INSERT de cabeçalho do pedido fora de transação @Transactional atômica — cabeçalho + itens juntos
3 Cancelamento sem validar status atual Máquina de estados com enum StatusPedido · 422 em transição inválida
4 Desconto hardcoded como 0 em todos os itens Campo desconto operacional no ItemPedidoRequestDTO
5 Estoque nunca atualizado ao criar ou cancelar pedido Decremento automático na criação · restituição no cancelamento
6 Status CONFIRMADO e FATURADO sem operação real no sistema PATCH /confirmar e PATCH /faturar implementados
7 Desativação de usuário irreversível (sem reativação) PATCH /reativar adicionado
8 Sem autenticação — acesso direto ao terminal do servidor JWT + Spring Security com 3 perfis de acesso
9 Sem API — integração com outros sistemas impossível REST completo com RFC 7807 Problem Details
