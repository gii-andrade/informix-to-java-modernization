Sistema de Pedidos — API Reference

Modernização Informix-4GL → Spring Boot 3  ·  Java 21  ·  JWT + BCrypt

Base URL: http://localhost:8080/api/v1

Auth

Único endpoint público. Retorna um JWT Bearer a ser usado em todos os demais endpoints.

POST /auth/login público Autenticar e obter token JWT ▶
Body (JSON)
CampoTipoDescrição
loginstringreqLogin do usuário cadastrado
senhastringreqSenha (mín. 8 chars)
Exemplo
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"login":"admin","senha":"password"}'
Resposta 200
{ "token": "eyJhbGci...", "perfil": "ADMIN", "expiresInMs": 86400000 }
Use o token em todas as chamadas seguintes: Authorization: Bearer <token>

Clientes

Porta cliente.4gl. GETs disponíveis para qualquer perfil autenticado. Escrita requer ADMIN ou OPERADOR.

GET /clientes autenticado Listar todos os clientes ▶ 
curl http://localhost:8080/api/v1/clientes \
  -H "Authorization: Bearer $TOKEN"
GET /clientes/{id} autenticado Buscar cliente por ID ▶ 
curl http://localhost:8080/api/v1/clientes/1 \
  -H "Authorization: Bearer $TOKEN"
POST /clientes OPERADOR+ Criar cliente → 201 Created ▶
Body
CampoTipoRegra
razaoSocialstringreqmáx 150 chars
nomeFantasiastringoptmáx 150 chars
documentostringreqúnico, máx 20 chars
emailstringoptformato e-mail, máx 120
telefonestringoptmáx 30 chars
enderecostringoptmáx 200 chars
cidadestringoptmáx 80 chars
ufstringoptexatamente 2 chars
cepstringoptmáx 10 chars
ativostringopt"S" ou "N" (default "S")
curl -X POST http://localhost:8080/api/v1/clientes \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "razaoSocial": "Empresa Exemplo Ltda",
    "documento":   "12.345.678/0001-90",
    "email":       "contato@exemplo.com",
    "cidade":      "São Paulo",
    "uf":          "SP",
    "ativo":       "S"
  }'
PUT /clientes/{id} OPERADOR+ Atualizar cliente completo ▶
Mesmos campos do POST. Substitui todos os dados do cliente.
curl -X PUT http://localhost:8080/api/v1/clientes/1 \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "razaoSocial": "Novo Nome S.A.", "documento": "12.345.678/0001-90", "uf": "RJ", "ativo": "S" }'
DELETE /clientes/{id} OPERADOR+ Excluir cliente → 204 / 409 se tiver pedidos ▶ 
curl -X DELETE http://localhost:8080/api/v1/clientes/1 \
  -H "Authorization: Bearer $TOKEN"
Retorna 409 Conflict se o cliente possuir pedidos vinculados.

Produtos

Porta produto.4gl. Inclui operações especializadas de preço e estoque ausentes no legado.

GET /produtos autenticado Listar todos os produtos ▶ 
curl http://localhost:8080/api/v1/produtos -H "Authorization: Bearer $TOKEN"
POST /produtos OPERADOR+ Criar produto → 201 Created ▶
CampoTipoRegra
codigostringreqúnico, máx 30 chars
descricaostringreqmáx 150 chars
unidadeMedidastringoptdefault "UN", máx 10
precoVendadecimalreq≥ 0, até 2 decimais
estoqueAtualdecimalopt≥ 0, até 3 decimais
ativostringopt"S" ou "N" (default "S")
curl -X POST http://localhost:8080/api/v1/produtos \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "codigo": "PROD-001", "descricao": "Caneta Esferográfica Azul",
    "unidadeMedida": "UN", "precoVenda": 2.50, "estoqueAtual": 100.000, "ativo": "S"
  }'
PATCH /produtos/{id}/preco?valor={valor} OPERADOR+ Atualizar preço de venda ▶ 
curl -X PATCH "http://localhost:8080/api/v1/produtos/1/preco?valor=3.99" \
  -H "Authorization: Bearer $TOKEN"
PATCH /produtos/{id}/estoque?valor={valor} OPERADOR+ Ajustar estoque atual ▶ 
curl -X PATCH "http://localhost:8080/api/v1/produtos/1/estoque?valor=250.000" \
  -H "Authorization: Bearer $TOKEN"
DELETE /produtos/{id} OPERADOR+ Excluir produto → 204 / 409 se tiver itens de pedido ▶ 
curl -X DELETE http://localhost:8080/api/v1/produtos/1 -H "Authorization: Bearer $TOKEN"

Pedidos

Porta pedido.4gl. Unifica cabeçalho + itens em uma única operação atômica. Implementa máquina de estados que não existia no legado.

Máquina de estados:
ABERTO──▶ /confirmar ──▶ CONFIRMADO──▶ /faturar ──▶ FATURADO  |  ABERTO ou CONFIRMADO ──▶ /cancelar ──▶ CANCELADO
POST /pedidos OPERADOR+ Criar pedido com itens → 201 Created ▶
CampoTipoDescrição
clienteIdlongreqID do cliente
usuarioIdlongreqID do usuário responsável
observacaostringoptmáx 500 chars
itens[].produtoIdlongreqProduto ativo
itens[].quantidadedecimalreq> 0, até 3 decimais
itens[].descontodecimalopt≥ 0 (default 0)
curl -X POST http://localhost:8080/api/v1/pedidos \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "clienteId": 1, "usuarioId": 1,
    "observacao": "Entrega urgente",
    "itens": [
      { "produtoId": 1, "quantidade": 10.000, "desconto": 0.00 },
      { "produtoId": 2, "quantidade":  2.000, "desconto": 5.00 }
    ]
  }'
GET /pedidos?status={status} autenticado Listar pedidos (todos ou filtrado por status) ▶ 
# Todos os pedidos
curl http://localhost:8080/api/v1/pedidos -H "Authorization: Bearer $TOKEN"

# Filtrar por status: ABERTO | CONFIRMADO | FATURADO | CANCELADO
curl "http://localhost:8080/api/v1/pedidos?status=ABERTO" -H "Authorization: Bearer $TOKEN"
PATCH /pedidos/{id}/confirmar OPERADOR+ Confirmar pedido (ABERTO → CONFIRMADO) ▶ 
curl -X PATCH http://localhost:8080/api/v1/pedidos/1/confirmar \
  -H "Authorization: Bearer $TOKEN"
Retorna 422 se o pedido não estiver no status ABERTO.
PATCH /pedidos/{id}/faturar OPERADOR+ Faturar pedido (CONFIRMADO → FATURADO) ▶ 
curl -X PATCH http://localhost:8080/api/v1/pedidos/1/faturar \
  -H "Authorization: Bearer $TOKEN"
PATCH /pedidos/{id}/cancelar OPERADOR+ Cancelar pedido (ABERTO ou CONFIRMADO → CANCELADO) ▶ 
curl -X PATCH http://localhost:8080/api/v1/pedidos/1/cancelar \
  -H "Authorization: Bearer $TOKEN"
O estoque dos itens é restituído automaticamente ao cancelar.
POST /pedidos/{id}/itens OPERADOR+ Adicionar item a pedido ABERTO ▶ 
curl -X POST http://localhost:8080/api/v1/pedidos/1/itens \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "produtoId": 3, "quantidade": 5.000, "desconto": 0.00 }'
DELETE /pedidos/{id}/itens/{produtoId} OPERADOR+ Remover item de pedido ABERTO ▶ 
curl -X DELETE http://localhost:8080/api/v1/pedidos/1/itens/3 \
  -H "Authorization: Bearer $TOKEN"

Usuários

Porta usuario.4gl. Todos os endpoints requerem perfil ADMIN. Perfis: ADMIN · OPERADOR · CONSULTA.

GET /usuarios?perfil={perfil} ADMIN Listar usuários (todos ou por perfil) ▶ 
# Todos
curl http://localhost:8080/api/v1/usuarios -H "Authorization: Bearer $TOKEN"

# Por perfil: ADMIN | OPERADOR | CONSULTA
curl "http://localhost:8080/api/v1/usuarios?perfil=OPERADOR" -H "Authorization: Bearer $TOKEN"
POST /usuarios ADMIN Criar usuário → 201 Created ▶
CampoTipoRegra
loginstringreqúnico · letras, números, . - _
nomestringreqmáx 120 chars
senhastringreqmín 8 chars — convertida em BCrypt
perfilenumoptADMIN | OPERADOR | CONSULTA (default OPERADOR)
curl -X POST http://localhost:8080/api/v1/usuarios \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "login": "joao.silva", "nome": "João Silva", "senha": "senha1234", "perfil": "OPERADOR" }'
PATCH /usuarios/{id}/desativar ADMIN Desativar usuário ▶ 
curl -X PATCH http://localhost:8080/api/v1/usuarios/2/desativar \
  -H "Authorization: Bearer $TOKEN"
PATCH /usuarios/{id}/reativar ADMIN Reativar usuário (lacuna corrigida — irreversível no legado) ▶ 
curl -X PATCH http://localhost:8080/api/v1/usuarios/2/reativar \
  -H "Authorization: Bearer $TOKEN"
PATCH /usuarios/{id}/senha ADMIN Alterar senha → 204 No Content ▶ 
curl -X PATCH http://localhost:8080/api/v1/usuarios/2/senha \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "senhaAtual": "senha1234", "senhaNova": "novaSenha99" }'
Exige confirmação da senha atual para evitar alteração não autorizada.

GET /relatorios/pedidos-por-status autenticado Pedidos agrupados por status com totais ▶ 
curl http://localhost:8080/api/v1/relatorios/pedidos-por-status \
  -H "Authorization: Bearer $TOKEN"
Resposta
[
  { "status": "ABERTO",     "quantidade": 3, "valorTotal": 540.00 },
  { "status": "CONFIRMADO", "quantidade": 1, "valorTotal": 200.00 },
  { "status": "FATURADO",   "quantidade": 5, "valorTotal": 1350.00 }
]
