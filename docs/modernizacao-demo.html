Demonstração da transformação do módulo de Pedidos do legado para a arquitetura Java.

✗ Módulo Legado (Informix)
  • 📄 Programa principal (4GL)
  • 📄 Módulo de entrada de pedido

  • 📄 Stored procedure: calc_total_pedido

✓ Módulo Modernizado (Spring Boot)
  • Entity Pedido, ItemPedido, Cliente
  • DTO PedidoRequestDTO, PedidoResponseDTO
  • Controller PedidoController
  • Service PedidoService
  • Repository PedidoRepository
  • Exception PedidoNotFoundException, ClienteInvalidoException
  • 🧪 PedidoServiceTest, PedidoControllerTest

Regras de Negócio Preservadas

✓ Validação de cliente ativo antes de criar pedido
✓ Inclusão e validação de itens no pedido (quantidade, produto, preço)
✓ Cálculo automático do valor total do pedido
✓ Atualização de estoque ao confirmar pedido
✓ Integridade referencial garantida via JPA + validações de domínio
✓ Regras de status e transições de estado do pedido

Benefícios da Modernização

Resultados alcançados com a migração para Java Spring Boot.

01

Código Organizado

Arquitetura em camadas com responsabilidades bem definidas e fácil navegação para desenvolvedores.

02

Facilidade de Manutenção

Alterações isoladas por camada sem impacto em cascata. Menor risco e menor tempo de entrega.

03

Maior Escalabilidade

Pronto para deploy em containers, cloud e arquiteturas distribuídas com escalabilidade horizontal.

04

Integração com Novas Tecnologias

APIs REST abertas, suporte a mensageria, autenticação moderna e conectores para qualquer sistema.

05

Redução de Esforço Futuro

Testes automatizados garantem regressão contínua. Menor custo de manutenção a longo prazo.

06

Padronização Arquitetural

Padrões reconhecidos pelo mercado (MVC, REST, JPA) facilitam onboarding de novos desenvolvedores.
